# DB Doctor — Claude Code Project Guide

AI-powered database diagnostics built for the Anthropic x Motier Hackathon. Uses Next.js 16 App Router, SQLite via `better-sqlite3`, and `@anthropic-ai/sdk` with `claude-opus-4-8`.

---

## Commands

```bash
npm run dev       # Start dev server at http://localhost:3000
npm run seed      # Create data/app.db with 50K orders (run once, or after deleting the DB)
npm run build     # Production build
npx tsc --noEmit  # Type-check without emitting
```

> **Never run `npm run seed` if `data/app.db` already has data** — the seed script is idempotent but seeding twice is wasteful. Check `data/` first.

---

## Project structure

```
app/
  page.tsx                  — All client UI: AgentTrace, ApproveModal, exportReport, AnimatedNumber
  layout.tsx                — Root layout, Geist font, page title
  globals.css               — Tailwind v4 import + all custom keyframe animations
  api/
    diagnose/route.ts       — POST: schema + EXPLAIN + timing → Claude → diagnosis + steps[]
    apply-fix/route.ts      — POST: CREATE INDEX → re-time → before/after speedup
    reset/route.ts          — POST: drops all non-system indexes on orders table

lib/
  db.ts                     — Process-level SQLite singleton, WAL mode

scripts/
  seed.ts                   — Creates tables + 50K rows. Deliberately omits index on orders.customer_id

data/                       — GITIGNORED. Generated by npm run seed.
  app.db

.env.local                  — GITIGNORED. Must contain ANTHROPIC_API_KEY=sk-ant-...
```

## Critical constraints

### Environment
- `ANTHROPIC_API_KEY` must live in `.env.local` — never hardcode it anywhere
- `data/` is gitignored — never commit the SQLite database
- `PRESENTATION.md` is gitignored — local-only pitch guide

### Next.js 16 specifics
- App Router only — no `pages/` directory
- API routes export named `GET`/`POST`/`PUT` functions, not a default export
- Server Components are the default — `'use client'` is required on `app/page.tsx`
- `serverExternalPackages: ['better-sqlite3']` in `next.config.ts` is mandatory — do not remove it

### Tailwind v4 class names (breaking changes from v3)
- Use `bg-linear-to-b` not `bg-gradient-to-b`
- Use `shrink-0` not `flex-shrink-0`
- Use `wrap-break-word` not `break-words`
- Run `npx tsc --noEmit` after UI changes to catch lint warnings from the PostToolUse hook

### better-sqlite3
- Synchronous API only — no `await` on DB calls
- All DB operations happen server-side (API routes) — never import `lib/db.ts` in client components
- The `getDb()` singleton is process-scoped — safe in Next.js dev (single process), fine for demo

### Anthropic SDK
- Model: `claude-opus-4-8` — do not downgrade
- Thinking: `{ type: 'adaptive' }` — do not disable
- Always use `client.messages.stream()` + `.finalMessage()` — not `client.messages.create()` — to avoid timeouts
- Response is JSON inside a text block — parse with `raw.match(/\{[\s\S]*\}/)` before `JSON.parse`

| Change the demo query | `SLOW_QUERY` constant in both `diagnose/route.ts` and `apply-fix/route.ts` |
| Add a new API route | Create `app/api/{name}/route.ts`, export `async function POST(req: NextRequest)` |

## The demo bug (intentional)

`scripts/seed.ts` creates the `orders` table **without** an index on `customer_id`. This forces a full table scan (`SCAN o` in EXPLAIN) on every customer order lookup. The agent detects this and proposes `CREATE INDEX idx_orders_customer_id ON orders(customer_id)`.

**Do not add an index to the seed script.** The missing index is the entire point of the demo.

To restore the bug after running the demo: click **Reset demo** in the UI, or `POST /api/reset`.

---

## Security rules

- `/api/apply-fix` **only** accepts `CREATE INDEX` and `CREATE UNIQUE INDEX` — reject everything else with a 400
- The user `question` field is used as prompt context only — never executed as SQL or shell
- Do not add any endpoint that runs arbitrary SQL from user input
- Do not add any endpoint that writes or deletes rows

---

## How the Claude call works

```
User question
    +
Schema (3 tables, column definitions)
    +
EXPLAIN QUERY PLAN output
    +
Query execution time (ms)
    ↓
claude-opus-4-8 (adaptive thinking, streaming)
    ↓
JSON: { diagnosis, root_cause, fix_sql, explanation, expected_improvement }
```

Claude is prompted as a "database performance expert" and instructed to return only JSON — no markdown fences, no extra prose. The response is parsed with a regex JSON extractor for robustness.
